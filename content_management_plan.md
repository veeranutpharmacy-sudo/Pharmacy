# แผนการจัดการและอัปเดตข้อมูลวิดีโอ (Content Management Plan)

เอกสารฉบับนี้จัดทำขึ้นเพื่อเป็นคู่มือและแผนรับมือ สำหรับการอัปเดตข้อมูลและคลิปวิดีโอที่จะได้รับจากทีมงานในอนาคต โดยระบบได้ถูกออกแบบมาให้รองรับความยืดหยุ่นในทุกกรณี

---

## 1. โครงสร้างการทำงานปัจจุบัน (Dynamic Rendering)
ระบบทำงานโดยการสร้าง "กล่องข้อความ" ตามจำนวนข้อมูลที่มีอยู่จริงในโค้ด (Array) 
- หากตึกนั้นมีข้อมูล **3 เรื่อง** ระบบจะสร้างกล่องขึ้นมา **3 กล่อง** เท่านั้น
- ระบบจะไม่สร้างกล่องเปล่าๆ ทิ้งไว้ให้เกิดพื้นที่ว่างที่ไม่ได้ใช้งาน

## 2. แผนรับมือกรณี "มีหัวข้อ แต่ยังไม่มีคลิป"
หากมีการกำหนดหัวข้อและรายละเอียดไว้แล้ว แต่ทีมงานยังตัดต่อคลิปไม่เสร็จ ระบบมีกลไกรองรับดังนี้:
- กล่องข้อความจะแสดงผลหัวข้อและรายละเอียดตามปกติ
- ในส่วนของวิดีโอ (ด้านบนของกล่อง) จะแสดงเป็น **พื้นที่สีฟ้า (Placeholder)** พร้อมข้อความว่า `"16:9 รองรับ YouTube"` เพื่อจองพื้นที่ไว้
- **วิธีแก้ไขเมื่อคลิปมาถึง:** เพียงแค่นำ YouTube ID มาใส่ในโค้ด ระบบจะเปลี่ยนพื้นที่สีฟ้าเป็นหน้าปกคลิปวิดีโอและสามารถกดเล่นได้ทันที

## 3. วิธีการอัปเดตข้อมูลคลิป (การใส่ YouTube ID)
เมื่อได้รับคลิปวิดีโอจากทีมงาน ให้นำคลิปไปอัปโหลดลงบน YouTube จากนั้นนำ "รหัสคลิป (YouTube ID)" มาใส่ในไฟล์ `index.html` (ประมาณบรรทัดที่ 1300 เป็นต้นไป)

**ตัวอย่างโครงสร้างข้อมูล:**
```javascript
1: {
  name: 'โรงพยาบาล Regional Health Coordination Center',
  icon: '🏥',
  sections: [
    {
      title: 'เรื่องที่ 1: ระบบจัดการข้อมูลผู้ป่วย',
      text: 'รายละเอียดการทำงานของระบบ...',
      youtube: 'XsNwcmQv0X' // นำรหัส 11 ตัวท้ายของ YouTube Link มาวางตรงนี้
    },
    {
      title: 'เรื่องที่ 2: ระบบที่ยังไม่มีคลิป',
      text: 'รายละเอียด...',
      // youtube: 'ยังไม่มีคลิป ไม่ต้องใส่บรรทัดนี้ ระบบจะแสดงกล่องสีฟ้าแทน'
    }
  ]
}
```

## 4. แผนรับมือกรณี "มีข้อความ แต่จะไม่มีคลิปวิดีโอเลย"
หากในอนาคตมีการตกลงกันว่า บางกล่องจะมีแค่การอธิบายด้วยตัวหนังสือ โดยจะไม่มีการใส่คลิปวิดีโอเลย สามารถปรับแต่งได้ดังนี้:
- ลบโค้ดส่วนของการแสดงพื้นที่สีฟ้า (Placeholder) ออกในไฟล์ `index.html` 
- กล่องข้อความจะหดตัวลงมา และแสดงผลเพียงแค่ หัวข้อ (Title) และ รายละเอียด (Description) ทำให้ดูสะอาดตาและไม่เหลือพื้นที่ว่าง

## 5. แผนการปรับผังหน้าจอ (Layout Rearrangement)
หากเมื่อนำคลิปทั้งหมดมารวมกันแล้ว พบว่าจำนวนคลิปในแต่ละตึกมีน้อยมาก (เช่น มีแค่ตึกละ 1-2 คลิป) การปล่อยให้แสดงผล 3 คอลัมน์อาจทำให้กล่องดูเล็กเกินไป สามารถปรับแก้ผังหน้าจอได้ง่ายๆ ดังนี้:

**เปลี่ยนการจัดเรียงจาก 3 คอลัมน์ เป็น 2 คอลัมน์ (ให้กล่องใหญ่ขึ้น)**
ค้นหาโค้ด CSS คำว่า `.modal-grid` แล้วปรับแต่ง:
```css
.modal-grid {
  display: grid;
  /* เปลี่ยนจาก repeat(3, 1fr) เป็น repeat(2, 1fr) เพื่อโชว์แค่ 2 กล่องต่อ 1 แถว */
  grid-template-columns: repeat(2, 1fr); 
  gap: 20px;
}
```

---
**สรุป:** ระบบถูกวางรากฐานไว้ให้ยืดหยุ่นสูงสุด การแก้ไขจำนวนกล่อง การเว้นพื้นที่คลิป หรือการปรับขนาดกล่อง สามารถทำได้ทันทีโดยไม่ต้องรื้อโครงสร้างหน้าเว็บใหม่ครับ
